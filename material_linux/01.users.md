# 🔹 Linux Users & Groups – 100/100 Notes

## 👤 What is a User in Linux?

In Linux, a user is an **"entity/individual account"** created for a specific person or a process to interact with the system.

Linux is a **multi-user system**, meaning multiple users can access the system and its resources simultaneously. In order to gain access, users must **log in (authenticate)**.

A user can interact with the system through:
- Terminal (CLI)
- Graphical User Interface (GUI)

Users can:
- Run commands
- Execute programs
- Access files
- Perform tasks based on their **permissions**

---

## 👑 Administrator Account: root

The **root user** is a special account with **unrestricted privileges**.

- UID: **0**
- Also called: **Superuser**

### Capabilities:
- Install/remove software
- Modify system configurations
- Manage users and groups
- Access all files without restriction

### 🔐 Security Note:
- Root has full control → **very dangerous if misused**
- Avoid direct root login
- Use **`sudo`** for controlled access

---

## 👥 Types of User Accounts

User accounts are classified into:

### 1. System User Accounts
- Created for services/daemons (e.g., `nginx`, `mysql`, `sshd`)
- Usually:
  - No login shell
  - Limited permissions

### 2. Regular / Standard User Accounts
- Created for human users
- Can log in and perform tasks
- Limited privileges

---

## 🆔 User ID (UID)

Each user is associated with a unique identifier called **UID (User ID)**.

### UID Ranges:
- UID **0** → root user  
- UID **1 – 999** → system users  
- UID **1000 – 60000** → regular users  

**Note:** Default UID & GID of root user = **0**

### 🔧 Useful Commands:

useradd username
